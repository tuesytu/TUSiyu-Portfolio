# 个人作品集网站 PRD

## 1. 项目概述

构建一个个人作品集网站，网站包含以下三大板块：

- 简历
- 作品集
- 其他作品

网站支持右上角中英文切换，默认语言为中文。

## 2. 项目目标

- 网站要清晰展示你的简历、项目作品和其他视觉产出
- 页面需要兼顾设计感与可读性
- 支持后续继续迭代视觉风格，不影响内容结构
- 适配手机、平板、桌面

## 3. 全局元素

### 3.1 语言切换

- 位置：页面右上角
- 交互：点击切换中文 / English
- 默认语言：中文
- 切换范围：所有文案内容

### 3.3 顶部主导航

当前使用导航文案：

- 中文：简历 / 作品 / 其他作品
- 英文：Resume / Works / Other Works

交互要求：

- 点击后可以作为页面切换或锚点导航
- 当前激活项高亮
- 切换过程应保持轻量顺滑

## 4. 页面结构

### 4.1 简历板块

- 竖向排布
- 包含：
  - 技能
  - 教育背景
  - 项目经历
  - 实习经历

### 4.2 作品集板块

- 展示三个主项目
- 每个项目包含：
  - 缩略图
  - 项目标题
  - 简短介绍
  - 查看详情入口

详情结构必须预留：

- 背景
- 角色
- 流程
- 图片
- 成果

### 4.3 其他作品板块

- 松散网格或档案式排列
- 仅展示图片和标题
- 点击可放大预览
- 不需要详情页

## 5. 内容范围

### 简历板块

- 技能
- 教育背景
- 项目经历
- 实习经历

### 作品集项目

- Pawards · 宠物丰容共创平台

### 其他作品

- 市政管道安装机器

- 建模渲染练习

## 6. 多语言要求

需要中英双语的内容包括：

- 顶部导航
- 简历板块标题
- 项目经历和实习经历
- 主项目标题与简介
- 其他作品标题
- 弹窗提示文字
- 页脚文案

实现方式：

- 前端映射或 i18n
- 无需后端

## 7. 非功能性需求

- 响应式设计：手机 / 平板 / 桌面
- 图片先允许使用占位图，后续替换
- 交互和动画应轻量
- 页面应方便后续继续做视觉风格精修

## 8. 推荐实现建议

- 当前可使用静态 HTML 版本快速预览
- 后续可迁移为 React + Vite 版本，便于扩展详情页和内容管理
